# 对齐报告 标题与必选章节

# 1 标题规范

一级标题固定：

- `# {主题} 对齐报告`

# 2 必选章节

- `# 1 摘要`
  - `## 1.1 结论`
  - `## 1.2 风险等级与影响面`
  - `## 1.3 建议动作`
- `# 2 审计范围与方法`
  - `## 2.1 审计范围`
  - `## 2.2 审计方法`
  - `## 2.3 证据口径`
- `# 3 差异表`
  - `## 3.1 差异表`
- `# 4 行动项`
  - `## 4.1 行动项清单`

裁决：

- “差异表”必须是表格，列至少包含：设计口径 / 代码现状 / 差异等级 / 风险 / 证据（文件路径）。
- “证据”必须是仓库相对路径，不得写本机绝对路径或本地 file scheme 链接。
- “行动项”必须可执行，包含负责人/优先级/验收条件。

# 3 可选章节

按需选用。

- `# 5 变更历史`
- `# 6 回归用例链接`
