# 接口规范 标题与必选章节

# 1 标题规范

一级标题固定：

- `# {模块} 接口规范`

# 2 必选章节

- `# 1 概述`
  - `## 1.1 本文档定义`
  - `## 1.2 术语与约定`
  - `## 1.3 关键约束`
- `# 2 核心接口`
  - `## 2.1 {接口名}` 可重复
    - `### 签名`
    - `### 参数说明`
    - `### 返回值`
    - `### 异常/错误码`
    - `### 线程安全性`
- `# 3 使用约束`
  - `## 3.1 生命周期与所有权`
  - `## 3.2 容量与预算`
  - `## 3.3 失败策略`
- `# 4 代码入口`
  - `## 4.1 接口定义位置`
  - `## 4.2 关键实现位置`
  - `## 4.3 相关测试与对齐文档`

裁决：

- 参数说明必须使用表格，列至少包含：参数名/类型/单位或取值范围/说明。
- 若接口有容量上限/预算/稳定序要求，必须写入“使用约束”，不得散落在正文叙述中。
- “代码入口”必须使用仓库相对路径；正文示例不得贴大段实现代码（单段不超过 30 行，可用伪代码替代）。
- 不得为历史调用方式保留静默兼容分支；如需迁移，必须通过版本化接口/显式开关/加载期强校验实现，并明确失败策略。

# 3 可选章节

按需选用。

- `### 示例`
- `# 性能与预算`
- `# 版本兼容性`
