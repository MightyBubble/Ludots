# 开发看板 标题与必选章节

# 1 标题规范

一级标题固定：

- `# {子系统} KANBAN`

# 2 必选章节

- `# 1 定位`
  - `## 1.1 本看板的唯一真相范围`
  - `## 1.2 不在本看板维护的内容`
- `# 2 使用规则`
  - `## 2.1 卡片字段规范`
  - `## 2.2 状态流转规范`
  - `## 2.3 WIP 上限与阻塞处理`
- `# 3 看板`
  - `## 3.1 Backlog`
  - `## 3.2 Doing`
  - `## 3.3 Review`
  - `## 3.4 Done`
- `# 4 里程碑`
  - `## 4.1 当前里程碑`
  - `## 4.2 下一个里程碑`
- `# 5 变更记录`

裁决：

- 每个子系统目录必须包含 `99_KANBAN.md`，作为当前开发计划的唯一真相。
- 任何开发计划与排期不得分散在其他文档正文中，只允许在其他文档中引用本看板的卡片 ID。
- 卡片必须包含可验收条款与证据入口，避免只写任务名。

# 3 可选章节

按需选用。

- `# 风险清单`
- `# 依赖与外部阻塞`
