# 测试用例 标题与必选章节

# 1 标题规范

一级标题固定：

- `# {用例名} 测试用例`

# 2 必选章节

- `# 1 测试目标`
  - `## 1.1 对应需求/裁决条款`
  - `## 1.2 覆盖范围`
- `# 2 前置条件`
  - `## 2.1 环境与版本`
  - `## 2.2 数据与初始状态`
- `# 3 测试步骤`
  - `## 3.1 步骤列表`
- `# 4 预期结果`
  - `## 4.1 可观察信号`
  - `## 4.2 判定标准`
- `# 5 回归关联`
  - `## 5.1 相关用例/缺陷/对齐报告链接`
- `# 6 自动化状态`
  - `## 6.1 自动化入口与触发方式`

裁决：

- “预期结果”必须能被读者验证，不写“看起来正常”。
- 如果用例验证的是裁决条款，必须在“回归关联”里指向对应裁决条款文档。
- “测试步骤”必须按序号列出，单步可执行，并明确观察点或检查点。

# 3 可选章节

按需选用。

- `# 数据集与种子`
- `# 性能指标`
- `# 失败诊断`
