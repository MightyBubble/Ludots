# 子文档路径与命名规范

# 1 裁决结论

结论优先。

- `docs/` 顶层按依赖顺序分层（00→07），层级目录内再按子系统拆分。
- 每个目录必须有入口文档 `00_总览.md`，作为该目录的唯一阅读入口。
- 每个子系统目录必须包含 `99_KANBAN.md`，作为当前开发计划的唯一真相。
- 子系统目录必须按类型分组布局。
- 所有可排序文件使用 `NN_标题.md`（两位数字），保证引用稳定。
- 旧路径必须保留时，使用“兼容入口”文档，正文只写新位置，不复述内容（仅用于文档路径兼容，不代表项目允许运行时向后兼容）。

# 2 目录规范

## 2.1 顶层分层目录

```
docs/
  00_文档总览/
  01_底层框架/
  02_核心引擎/
  03_基础服务/
  04_游戏逻辑/
  05_高级逻辑/
  06_交互层/
  07_附录与扩展/
```

## 2.2 子系统目录的入口与索引

每个子系统目录至少包含：

- `00_总览.md`：目录定位、SSOT、阅读顺序、子文档索引
- `99_KANBAN.md`：当前开发计划的唯一真相

# 3 子文档布局规范

## 3.1 按类型分组布局

```
{子系统目录}/
  00_总览.md
  99_KANBAN.md
  01_架构设计/
    01_主题A.md
  02_接口规范/
    01_主题A.md
  03_配置结构/
    01_主题A.md
  04_裁决条款/
    01_主题A.md
  05_对齐报告/
    01_主题A.md
  06_测试用例/
    01_主题A.md
  07_使用手册/
    01_主题A.md
```

裁决：

- 子系统根目录只允许 `00_总览.md` 与 `99_KANBAN.md`。
- 不允许出现“同一类型的文档分散在多个目录”的情况；需要拆分时，按主题拆目录，而不是按作者拆目录。

# 4 文件命名规范

## 4.1 入口文档

- 统一命名：`00_总览.md`
- 仅 `docs/00_文档总览/` 允许额外存在 `00_README.md` 作为全局入口。

## 4.2 排序文件

强制。

- 命名：`NN_标题.md`
- 规则：`NN` 为两位数字（`01`～`99`），同目录唯一。
- 稳定性：发布后尽量不改 `NN`，避免引用失效；需要调整排序时优先用新增文件承接内容。

## 4.3 兼容入口

强制，不允许留兼容。

# 5 禁止事项

- 空目录（目录必须有 `00_总览.md`）。
- 用日期作为排序号（例如 `20260205_...`）。
- 同目录重复 `NN`，或无序号散文件。
- 在正文中写本机绝对路径或 file scheme 本地链接。
