# 配置结构 标题与必选章节

# 1 标题规范

一级标题固定：

- `# {配置名} 配置结构`

# 2 必选章节

- `# 1 概述`
  - `## 1.1 本文档定义`
  - `## 1.2 使用场景与边界`
  - `## 1.3 关联运行时消费点`
- `# 2 文件位置与命名`
  - `## 2.1 目录位置`
  - `## 2.2 命名规则`
  - `## 2.3 发现与加载机制`
- `# 3 完整示例`
  - `## 3.1 最小可用示例`
  - `## 3.2 常见变体`
- `# 4 字段说明`
- `# 5 校验规则`
  - `## 5.1 加载期强校验`
  - `## 5.2 错误信息与诊断`
- `# 6 版本兼容性`
  - `## 6.1 版本号策略`
  - `## 6.2 破坏性变更与迁移方式`

裁决：

- “字段说明”必须用表格，列至少包含：字段/类型/必填/默认值/说明。
- “校验规则”必须描述加载期强校验与失败策略（fail-fast），不得只写“应当/建议”。
- 默认不做向后兼容与静默 fallback：破坏性变更必须提升版本并提供迁移方式；加载期必须强校验并 fail-fast，不得悄悄接受旧字段/旧格式。

# 3 可选章节

按需选用。

- `# 合并规则`
- `# 示例：常见错误与诊断信息`
- `# 与 Mod/VFS 的关系`
