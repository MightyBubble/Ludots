# 外部参考 标题与必选章节

# 0 裁决结论

- 一个插件/库一篇文档：外部参考以“单插件/单库”为最小粒度，不把多个插件混在一篇正文里。
- 必须讲清楚：架构形态（模块/数据流/线程模型）、技术选型（依赖/约束/取舍）、参考价值（优缺点与可吸收点）。
- 外部参考不得成为项目口径真源；若需要形成项目口径，必须回填到对应子系统 SSOT 文档。

# 1 标题规范

一级标题固定：

- `# {项目/库名} 外部参考`

# 2 必选章节

- `# 1 项目概述`
  - `## 1.1 项目定位与要解决的问题`
  - `## 1.2 版本与关键依赖`
- `# 2 架构与技术选型`
  - `## 2.1 总体架构与模块边界`
  - `## 2.2 核心数据结构与关键流程`
  - `## 2.3 并发模型与生命周期（可选：线程/Job/安全点）`
  - `## 2.4 技术选型与约束（依赖、平台耦合、性能假设）`
- `# 2 核心能力矩阵`
  - `## 2.1 能力点与映射表`
- `# 3 适用性评估`
  - `## 3.1 适配成本与风险`
  - `## 3.2 与现有实现差异`
- `# 3 参考价值（必须明确优缺点）`
  - `## 3.1 值得吸收的点`
  - `## 3.2 不适合/不建议引入的点`
  - `## 3.3 对 Ludots 的落地点（回填清单）`
- `# 4 结论与建议`
  - `## 4.1 是否引入`
  - `## 4.2 后续动作`

裁决：

- “核心能力矩阵”至少包含：能力点 / 该项目支持情况 / 对 Ludots 的映射 / 风险与成本。
- 外部参考的归属位置是对应子系统目录，统一放在 `{子系统目录}/07_外部参考/` 下。
- `docs/07_附录与扩展/external/` 仅承载全项目外部参考入口索引，不承载正文（正文必须留在子系统目录）。
- 插件对比类文档可作为补充，但不替代单插件外部参考正文。

# 3 可选章节

按需选用。

- `# 代码阅读笔记`
- `# 迁移计划与里程碑`
- `# 与现有实现对齐报告链接`
