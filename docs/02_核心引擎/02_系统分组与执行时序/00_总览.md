# 系统分组与执行时序 总览

# 1 本目录定位

## 1.1 读者与范围

读者：

- 引擎/基建开发者：需要固化 Phase（SystemGroup）顺序与暂停/恢复语义。
- 上层系统开发者：需要知道系统应落在哪个 Phase，以及不允许做哪些跨 Phase 依赖。

范围：

- SystemGroup/Phase 的固定顺序与一致性视图
- 与 Pacemaker/TimeSlice 的关系（引用 SSOT，不复述）

## 1.2 本目录边界

- 只描述“顺序与职责边界”，不描述具体业务系统实现。  
- TimeSlice 的多频率策略细节由 `docs/02_核心引擎/03_TimeSlice与分帧策略/00_总览.md` 承载。  

## 1.3 推荐阅读顺序

1. `docs/02_核心引擎/02_系统分组与执行时序/01_系统分组与执行时序_架构设计.md`
2. `docs/02_核心引擎/03_TimeSlice与分帧策略/00_总览.md`

# 2 SSOT

## 2.1 代码入口

- PhaseOrder 与暂停/恢复游标：`src/Core/Engine/Pacemaker/PhaseOrderedCooperativeSimulation.cs`
- 系统注册与分组落位：`src/Core/Engine/GameEngine.cs`

## 2.2 口径说明

- Phase 顺序以 cooperative simulation 的 `PhaseOrder` 为准；文档只做解释与边界裁决。  

## 2.3 关键约束

1. 顺序固定：SystemGroup 不允许运行态重排。  
2. 一致性视图：禁止跨 Phase 偷写共享状态导致可见性不一致。  
3. 预算口径统一：长任务必须 time-slice，可熔断，可恢复。
