# 总架构 KANBAN

# 1 定位
## 1.1 本看板的唯一真相范围
- 本文档是总架构子系统的当前开发计划唯一真相。
- 任何计划拆解、优先级与排期只在这里维护，其它文档只引用卡片 ID。

## 1.2 不在本看板维护的内容
- 架构口径与裁决条款。
- 接口细节与配置字段说明。

# 2 使用规则
## 2.1 卡片字段规范
- ID：子系统内唯一，格式为 `K-001` 递增。
- 卡片：一句话描述要交付的结果。
- 负责人：明确到人。
- 验收标准：可判定，给出验证方法。
- 证据入口：仓库相对路径，指向代码或测试或对齐报告。

## 2.2 状态流转规范
- Backlog -> Doing -> Review -> Done
- Doing 超过 3 天未推进，必须标记阻塞原因并给出下一步动作。

## 2.3 WIP 上限与阻塞处理
- Doing 列最多 5 张卡片。
- 超过上限时，必须先收敛 Doing，再拉入新卡片。
