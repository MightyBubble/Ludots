# 基础服务 总览

# 1 本目录定位

本目录承载在引擎之上、被上层大量复用的基础服务：Graph 运行时、脚本与事件、空间服务、物理系统。

## 1.1 读者与范围

读者：

- 引擎/基建开发者：需要定义“领域无关”的可复用服务口径与边界。
- 上层系统开发者：需要知道如何依赖这些服务（接口/SSOT），以及哪些行为是禁止的。

范围：

- Graph 运行时：确定性执行、可 time-slice 的图程序运行时
- 脚本与事件：脚本上下文、事件键、Trigger 执行框架与 fail-fast 约束
- 空间服务：空间索引、查询契约、稳定序、固定容量与 dropped 退化口径
- 物理系统：物理后端能力与预算口径（当前以 2D 为主）

## 1.2 本目录边界

- 本目录只承载“基础服务”层，不承载核心引擎生命周期与调度（见 `docs/02_核心引擎/`）。  
- 外部参考不在本层沉淀，必须归档到 `docs/07_附录与扩展/external/`。  

## 1.3 推荐阅读顺序

1. `docs/02_核心引擎/00_总览.md`（预算/确定性/fail-fast 总口径）
2. `docs/03_基础服务/06_空间服务/00_总览.md`（空间查询契约先于物理/避障消费）
3. `docs/03_基础服务/04_Graph运行时/00_总览.md`
4. `docs/03_基础服务/05_脚本与事件/00_总览.md`
5. `docs/03_基础服务/07_物理系统/00_总览.md`

# 2 SSOT

## 2.1 代码入口

- GraphRuntime：`src/Core/GraphRuntime/`
- 脚本与事件：`src/Core/Scripting/`
- 空间服务：`src/Core/Spatial/`
- 物理系统：`src/Core/Physics/`

## 2.2 口径说明

- 预算与暂停/恢复语义以引擎 cooperative simulation 为准：`src/Core/Engine/Pacemaker/PhaseOrderedCooperativeSimulation.cs`。  
- 固定容量/稳定序/dropped 的裁决条款由各服务域 SSOT 文档给出。  

## 2.3 关键约束

1. 禁止领域语义渗透：基础服务不得引入 GAS/AI/MOBA 等业务词与策略分支。  
2. 确定性优先：排序/tie-break 必须固化，不依赖容器遍历顺序。  
3. 可预算：任何高成本路径必须可 time-slice 或具备固定容量输出与 dropped 可观测。
