# 物理系统 总览

# 1 本目录定位

本目录定义物理系统（当前以 2D 为主）的边界、数据结构与预算口径。
阅读顺序上放在“空间服务”之后：先统一坐标域与空间查询契约，再描述物理如何作为后端/消费者接入（宽相、固定容量查询、dropped 可观测）。

## 1.1 读者与范围

读者：

- 基建开发者：实现物理后端与空间查询契约的对齐。
- 上层系统开发者：以可预算的方式使用查询能力，并制定 dropped 退化策略。

范围：

- PhysicsWorld：最小查询后端与数据结构
- Physics2D Ticking：targetHz/MaxStepsPerFixedTick 等更新策略口径

## 1.2 本目录边界

- 不提供完整动力学求解器与约束解算（当前只提供最小索引/查询能力）。  
- 预算与 time-slice 的通用语义引用核心引擎 SSOT，不在本目录复述。  

## 1.3 推荐阅读顺序

1. `docs/03_基础服务/06_空间服务/00_总览.md`
2. `docs/02_核心引擎/03_TimeSlice与分帧策略/00_总览.md`
3. `docs/03_基础服务/07_物理系统/01_Physics2D架构.md`
