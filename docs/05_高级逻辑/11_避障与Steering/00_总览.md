# 避障与 Steering 总览

# 1 本目录定位

## 1.1 读者与范围

本目录定义近场避障、Steering 行为、群体引导，以及基于 TimeSlice 的预算推进与退化策略。
本目录依赖空间服务与导航寻路，禁止引入业务语义；所有可扩展语义通过 token/ID 注入。

## 1.2 本目录边界

- 覆盖范围：近场避障/steering 的架构与裁决口径（更新频率、退化策略、与空间查询的契约关系）。  
- 不覆盖：空间服务接口本体、核心引擎 PhasePipeline/TimeSlice 的通用机制（本目录只引用 SSOT）。

# 2 SSOT

## 2.1 代码入口

- 空间查询门面与稳定序：`src/Core/Spatial/SpatialQueryService.cs`、`src/Core/Spatial/SpatialQueryPostProcessor.cs`  
- 物理系统的 targetHz 分发参考：`src/Core/Engine/Physics2D/DiscreteRateTickDistributor.cs`  

## 2.2 口径说明

- 近场避障的候选集必须来自空间服务的固定容量输出（Span），并按“稳定序 + 去重”口径消费。  
- 任何降频/分帧策略必须以离散 tick 为裁决真源，禁止以 wall-clock 浮点 dt 驱动裁决。  

## 2.3 关键约束

1. 禁止业务语义：不出现“方阵/精英”等业务词；差异通过 token/ID 与配置表达。  
2. 禁止静默 fallback：dropped/越界/缺失依赖必须可观测并显式退化。  
3. Non‑Cullable 与 Cullable 必须区分：裁决级结果不可被镜头/裁剪影响。
