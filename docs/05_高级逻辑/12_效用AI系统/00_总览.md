# 效用AI系统 总览

# 1 本目录定位
## 1.1 读者与范围

适用读者：

- 需要在 Ludots 中实现大规模群体行为/索敌/目标选择的系统设计与实现者
- 需要为 UGC 定义可扩展的“目标选择输入/配置/输出”约束的内容设计者

范围：

- 本目录提供效用AI系统的“统一口径与约束”，不引入任何业务语义；所有语义必须通过 token/ID 注入（AttributeId/TagId/OrderTagId/BlackboardKeyId/CurveId）。

## 1.2 本目录边界

- 本目录不定义具体单位业务（不出现任何业务字眼），不承担技能/效果执行细节。
- 本目录不绑定任何具体空间实现（AOI/物理/导航网）；空间输入通过适配层接口获取。
- 本目录的裁决输出以 Order 为真源（Order-first），由能力系统（GAS）消费与执行。

## 1.3 推荐阅读顺序

1. `01_架构设计/00_总览.md` → `01_效用AI系统_架构设计.md` → `02_效用AI_架构设计.md`
2. `02_接口规范/00_总览.md` → `01_索敌查询.md` → `02_空间查询适配层.md`
3. `03_配置结构/00_总览.md` → `03_响应曲线_配置结构.md` → `02_效用AI配置_配置结构.md` → `01_评分规范.md`

# 2 SSOT
## 2.1 代码入口

- `src/Core/Gameplay/GAS/Orders/OrderQueue.cs`
- `src/Core/Gameplay/GAS/Orders/OrderArgs.cs`

## 2.2 口径说明

- 效用AI系统在“目标选择/决策”上只负责产出 Order（裁决输出）；Order 入队后即成为玩法裁决真源。
- 后续“如何执行 Order/如何映射到技能与效果”的口径以 GAS 为准，本目录不复述。

## 2.3 关键约束

- 固定上界：候选上限固定为 16；所有阶段必须可预算化、可熔断、可审计。
- 零 GC：运行态禁止字符串解析与托管集合飞线；输入/输出必须为值类型快照。
- 确定性：收集/排序/采样使用 StableKey 与固定 tie-break，不依赖遍历顺序。
- 频率口径：效用AI系统的计算频率必须由离散 Clock 域驱动，支持回合制手动 step。
